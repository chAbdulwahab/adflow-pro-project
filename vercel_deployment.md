# 🚀 Vercel Deployment Guide: AdFlow Pro

This guide outlines the steps to deploy **AdFlow Pro** to Vercel and configure the necessary environment variables and automated workflows.

## 1. Environment Variables Configuration

In your Vercel Project Settings, add the following environment variables. You can copy most of these from your `.env.local`.

| Variable | Description | Source / Example |
| :--- | :--- | :--- |
| `NEXT_PUBLIC_SUPABASE_URL` | Your Supabase project URL | `.env.local` |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Public anon key for client-side use | `.env.local` |
| `SUPABASE_SERVICE_ROLE_KEY` | **Secret** key for admin database operations | `.env.local` |
| `JWT_SECRET` | Secret key for custom auth and manual cron triggers | `.env.local` |
| `CRON_SECRET` | **Required** for Vercel Cron Jobs protection | Auto-generated by Vercel or custom string |
| `APP_URL` | The production URL of your app | `https://your-app.vercel.app` |

> [!IMPORTANT]
> Ensure `NODE_ENV` is set to `production` in Vercel.

## 2. Automated Cron Jobs

The project is configured with `vercel.json` to handle automated tasks:

1.  **Publish Scheduled Ads**: Runs every hour (`0 * * * *`).
2.  **Expire Ads**: Runs every hour at 30 minutes (`30 * * * *`).

### Manual Trigger
You can manually trigger these jobs by sending a `POST` or `GET` request to the endpoints with the `x-cron-secret` header set to your `JWT_SECRET`.

Example:
```bash
curl -X GET https://your-app.vercel.app/api/cron/publish-scheduled \
  -H "x-cron-secret: your_jwt_secret"
```

## 3. Deployment Steps

1.  **Push to GitHub**: Ensure all changes (including the new `vercel.json`) are pushed to your repository.
2.  **Import to Vercel**: Connect your GitHub repo to Vercel.
3.  **Configure Build**:
    *   **Framework Preset**: Next.js
    *   **Root Directory**: `./` (or just leave default)
    *   **Build Command**: `npm run build`
4.  **Add Envs**: Paste the variables listed above.
5.  **Deploy**: Click Deploy!

## 4. Troubleshooting

*   **Auth Issues**: Ensure `JWT_SECRET` matches between your local test and production.
*   **Database Connection**: If Supabase fails, verify the `SUPABASE_SERVICE_ROLE_KEY` is the **Service Role** key, not the Anon key.
*   **Cron Failures**: Check Vercel Logs under the "Logs" tab and filter by "Cron Jobs".
